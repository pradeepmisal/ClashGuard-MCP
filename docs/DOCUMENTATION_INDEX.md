# Documentation Index

This directory contains the primary technical, compliance, and setup documents for ClashGuard.

## Core documents

- `PRD.md` — Product requirements and execution plan.
- `IMPLEMENTATION_SUMMARY.md` — High-level implementation overview and status.
- `CLASH_VISUALIZATION_TECHNICAL_GUIDE.md` — Revit visualization design and implementation details.
- `REVIT_CLASH_VISUALIZATION_PLAN.md` — Revit clash visualization workflow and architecture.
- `PRODUCTION_CONFIG_REFERENCE.md` — Production configuration options and environment settings.
- `PRODUCTION_DEPLOYMENT_GUIDE.md` — Deployment guidance for the MCP backend.
- `SETUP_GUIDE.md` — Setup instructions for local development and demo.
- `SETUP_COMPLETE.md` — Completion checklist and artifact validation.
- `QUICK_REFERENCE.md` — Quick start references and common commands.
- `START_HERE.txt` — Initial onboarding instructions for the repository.
- `AUTODESK_COMPLIANCE.md` — Marketplace compliance summary.
- `AUTODESK_PUBLISHER_DECLARATION.md` — Publisher declaration and compliance notes.
- `INTEGRATION_CHANGES.md` — Revit integration and code change log.
- `PROJECT_PLAN.md` — Execution plan and product requirements document extracted from legacy notes.

## Security and architecture docs

- `ARCHITECTURE.md` — Active component and folder architecture for the codebase.
- `SECURITY.md` — Security guidance for production deployment and code hygiene.

## How to use this folder

1. Open `README.md` in the repository root.
2. Review `SECURITY.md` before editing code or committing secrets.
3. Use `docs/DOCUMENTATION_INDEX.md` as the single source of truth for all project documents.
